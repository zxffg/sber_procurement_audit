# Внутренний аудит ИТ-закупок группы Сбер (2024–2025)

## Описание проекта
Проект — тестовое задание на позицию Data Analyst. Цель проекта — проведение аудита закупочной деятельности 10 ключевых дочерних зависимых обществ (ДЗО) ИТ-контура ПАО Сбербанк на основе данных с ЭТП SberB2B. 

Реализован полный цикл аналитики: сбор данных, проектирование реляционной модели данных, статистический поиск аномалий и интеграция локальных алгоритмов машинного обучения для работы с неструктурированным текстом.

## Проблематика и ограничения выборки
При сборе и анализе данных учитывались строгие регуляторные ограничения российского рынка:
1. **ПП РФ №301 (Засекречивание контрагентов):** Дочерние общества санкционных банков скрывают ИНН и наименования победителей тендеров. Для обхода этого ограничения в проекте реализовано LLM-профилирование на базе локальной LLM.
2. **Закрытие API ЕИС:** С 1 января 2025 года публичный доступ к API Единой информационной системы ограничен. Данные собирались путем парсинга коммерческой площадки SberB2B.

## Стек технологий
* **Язык программирования:** Python 3.10+
* **Обработка и анализ данных:** Pandas, NumPy
* **Базы данных:** SQLite (in-memory), SQL
* **Визуализация:** Seaborn, Matplotlib
* **Machine Learning / LLM:** Ollama, Qwen 2.5 (3B) — локальный инференс


## Этапы реализации

### Этап 1. Очистка и предобработка данных (ETL)
* Загрузка сырого массива (1 859 записей).
* Дедупликация процедур по уникальным ключам.
* Обработка пропусков и нормализация типов данных (приведение дат, конвертация валют).

### Этап 2. Проектирование базы данных
* Инициализация in-memory СУБД SQLite для обеспечения работы проекта "из коробки".
* Написание SQL-запросов для агрегации данных по месяцам, компаниям и типам закупок.

### Этап 3. Аналитический модуль и Машинное обучение
* **Поиск аномалий:** Внедрен статистический алгоритм (Z-score) для выявления экстремальных ценовых выбросов. Порог чувствительности адаптирован под малые выборки и установлен на уровне 2.5 стандартных отклонений внутри групп заказчиков.
* **LLM-профилирование:** Локальная Qwen 2.5 3B классифицирует поставщиков по масштабу бизнеса (Tier-1 / Tier-2) на основе справочника CNews/TAdviser. Данные не покидают периметр.
* **Корреляционный анализ:** Проведен анализ влияния макроэкономики (Ключевая ставка ЦБ РФ) на объемы ИТ-закупок.

### Этап 4. Визуализация и бизнес-инсайты
* Построены графики динамики освоения бюджета, структуры закупок и корреляционные матрицы.
* Исключены "шумы" (отмененные и несостоявшиеся тендеры).
* * Сформулированы бизнес-выводы: сезонность, CAPEX-доминирование, макроэкономическая неэластичность.

## Пайплайн воспроизведения проекта

### Подготовка окружения
1. Убедитесь, что у вас установлен **Python 3.10+**.
2. Склонируйте репозиторий с проектом на локальную машину и перейдите в папку проекта:
   ```bash
   git clone https://github.com/zxffg/sber_procurement_audit
   ```
3. Откройте терминал в корневой папке проекта и установите все необходимые зависимости:
   ```bash
   pip install -r requirements.txt
   ```
