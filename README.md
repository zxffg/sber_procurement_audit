# Внутренний аудит ИТ-закупок группы Сбер (2024–2025)

## 📌 Описание проекта
Данный проект представляет собой техническое задание на позицию Data Analyst. Цель проекта — проведение аудита закупочной деятельности 10 ключевых дочерних зависимых обществ (ДЗО) ИТ-контура ПАО Сбербанк на основе данных с ЭТП SberB2B. 

В рамках работы реализован полный цикл аналитики: сбор данных, проектирование реляционной модели данных, статистический поиск аномалий и интеграция локальных алгоритмов машинного обучения для работы с неструктурированным текстом.

## Проблематика и ограничения выборки
При сборе и анализе данных учитывались строгие регуляторные ограничения российского рынка:
1. **ПП РФ №301 (Засекречивание контрагентов):** Дочерние общества санкционных банков скрывают ИНН и наименования победителей тендеров. Для обхода этого ограничения в проекте реализован RAG-пайплайн на базе локальной LLM.
2. **Закрытие API ЕИС:** С 1 января 2025 года публичный доступ к API Единой информационной системы ограничен. Данные собирались путем парсинга коммерческой площадки SberB2B.

* **Базы данных:** SQLite (in-memory), SQL

## Этапы реализации

### Этап 1. Очистка и предобработка данных (ETL)
* Загрузка сырого массива (1 859 записей).
* Дедупликация процедур по уникальным ключам.
* Обработка пропусков и нормализация типов данных (приведение дат к ISO 8601, конвертация валют).

### Этап 2. Проектирование базы данных
* Инициализация in-memory СУБД SQLite для обеспечения работы проекта "из коробки" (Zero-Setup).
* Перевод колонок в snake_case, создание таблицы `procurement_flat`.
* Написание SQL-запросов для агрегации данных по месяцам, компаниям и типам закупок.

### Этап 3. Аналитический модуль и Машинное обучение
* **Поиск аномалий:** Внедрен статистический алгоритм (Z-score) для выявления экстремальных ценовых выбросов, превышающих 3 стандартных отклонения внутри групп заказчиков.
* **LLM & RAG-пайплайн:** Развернута локальная модель Qwen 2.5. Алгоритму передан справочник ИТ-рынка (Tier-1 / Tier-2) для вероятностного определения профиля скрытых поставщиков на основе текстов ТЗ. Полностью соблюден контур информационной безопасности (данные не отправлялись во внешние API).
* **Корреляционный анализ:** Проведен анализ влияния макроэкономики (Ключевая ставка ЦБ РФ) на объемы ИТ-закупок.

### Этап 4. Визуализация и бизнес-инсайты
* Построены графики динамики освоения бюджета, структуры закупок и корреляционные матрицы.
* Исключены "шумы" (отмененные и несостоявшиеся тендеры).
* Сформулированы обоснованные бизнес-выводы в формате Executive Summary.
