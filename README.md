## Этап 1. Сбор данных

Пперед сбором данных, мною был составлен полный список юр.лиц, которые относятся к группе компании Сбер. Вы можете ознакомиться с ним в файле `jupyter`.

### Обоснование выбора вертикали

Для анализа выбран **ИТ/финтех-контур экосистемы Сбера** — дочерние общества, работающие в сфере технологий, цифровых финансов и сервисов. Выбор обусловлен тремя факторами:

1. **Активность закупок.** ИТ-компании закупают ПО, оборудование, облачные мощности регулярно и в значительных объёмах, что даёт статистически достаточную выборку.
2. **Аналитическая ценность.** Закупки ИТ-сектора тесно коррелируют с внешними факторами (курс USD/RUB, ключевая ставка), что позволяет строить содержательные гипотезы в Этапе 3.
3. **Доступность данных.** В отличие от головного ПАО Сбербанк (данные закрыты с 2022 г. по регуляторным основаниям), дочерние ИТ-структуры публикуют закупки в коммерческой секции ЭТП.

Для составления полного перечня юридических лиц группы использован **Центр раскрытия корпоративной информации Интерфакс** ([e-disclosure.ru](https://e-disclosure.ru)), предоставляющий данные в открытом доступе, поскольку отчётность по МСФО на официальном сайте Сбера содержит ограниченные сведения о составе группы в условиях санкционного давления.

### Площадки из технического задания — результаты анализа

**[Сбербанк-АСТ](sberbank-ast.ru) с поддоменом `utp-` Универсальная торговая площадка - основной источник.**
[ЕИС](zakupki.gov.ru), [РТС-тендер](rts-tender.ru), [Росэлторг](roseltorg.ru), [ТекТорг](tektorg.ru), [ЗаказРФ](etp.zakazrf.ru), [ЛотОнлайн](lot-online.ru), [ЕТП ГПБ](etpgpb.ru).

Большинство площадок из перечня задания не содержат актуальных данных по группе Сбер за 2024–2025 гг. по двум независимым причинам.

**Постановление Правительства РФ №301 от 06.03.2022**
С марта 2022 года организации, находящиеся под санкциями, **освобождены от обязательного размещения информации о закупках в ЕИС**. Поскольку ПАО Сбербанк находится под западными санкциями, сведения о его закупках в публичном реестре ЕИС за 2024–2025 гг. **отсутствуют полностью.** Данное ограничение существенно снижает прозрачность закупочной деятельности крупнейшего банка страны с государственным участием.

**Ограничение доступа к данным ЕИС с 01.01.2025**
С 1 января 2025 года **массовая выгрузка данных из ЕИС** стала возможна исключительно через **авторизованный API** вместо прежнего открытого FTP-сервера. Ранее свободный доступ позволял аналитическим агрегаторам и исследователям получать данные без регистрации. Теперь для получения массива данных необходима регистрация в ЕИС как юридическое лицо или ИП.

Агрегаторы (tenderer.ru и др.) прекратили полноценное обновление данных с начала 2025 г. Данные за этот период доступны только через прямой запрос к API ЕИС (нужна регистрация). Коммерческие сервисы по типу [Контур.Закупки](https://zakupki.kontur.ru) не несут практической ценности.

**Шаг 3. Консолидация файлов**
```python
# Объединение файлов по всем компаниям
import pandas as pd, glob

files = glob.glob('data/raw/*.xlsx')
dfs = []
for f in files:
    df = pd.read_excel(f)
    dfs.append(df)

df_all = pd.concat(dfs, ignore_index=True)
df_all.to_excel('data/processed/all_purchases_raw.xlsx', index=False)
```

**Шаг 4. Очистка и дедупликация**
```python
# Дедупликация по уникальному коду процедуры
df_clean = df_all.drop_duplicates(subset=['Код процедуры'])
# Статистика: до 1859 строк → уникальных процедур меньше
# (многолотовые закупки дают N строк на 1 процедуру)
```

**Шаг 5. Обезличивание персональных данных**
```python
import re
# Маскирование email и телефонов в описаниях лотов
df_clean['Наименование предмета закупки'] = df_clean[
    'Наименование предмета закупки'
].apply(lambda x: re.sub(r'[\w.+-]+@[\w-]+\.[a-z]{2,}', '[EMAIL]', str(x)))
df_clean['Наименование предмета закупки'] = df_clean[
    'Наименование предмета закупки'
].apply(lambda x: re.sub(r'(\+7|8)[\s\-]?\(?\d{3}\)?[\s\-]?\d{3}[\s\-]?\d{2}[\s\-]?\d{2}', '[PHONE]', str(x)))
```

### Известные технические ограничения

| Проблема | Описание | Решение |
|---|---|---|
| JavaScript-пагинация | Кнопка «Далее» на сайте не поддаётся стандартным скраперам | Ручная итерация по страницам |
| Лимит строк | Бэкенд Сбербанк-АСТ блокирует попытки выгрузки >500 строк за раз | Постраничная выгрузка |
| Многолотовые закупки | 1 процедура = N строк по числу лотов | Дедупликация по `Код процедуры` |
| Версионирование | Система сохраняет версии изменённых процедур | Берём последнюю версию |
| Нулевые суммы | 899 из 1859 записей имеют НМЦ = 0 | Рамочные договоры; анализируется отдельно |

### Типология данных (два класса процедур)

В ходе анализа выявлены два различных типа процедур:

- **Закупочная деятельность (Procurement)** — приобретение товаров, работ, услуг для обеспечения деятельности компаний
- **Реализация активов (Asset Liquidation)** — продажа имущества через Сбербанк-АСТ (121 запись)

Для корректного анализа объёмов закупок эти классы **разделены** во всех последующих расчётах.

## 6. Ключевые гипотезы для Этапа 3

| # | Гипотеза | Данные | Метод проверки |
|---|---|---|---|
| H1 | Рост закупок ИТ-оборудования у SberCloud коррелирует с динамикой курса USD/RUB | Закупки Облачных технологий + данные ЦБ РФ | Корреляция Пирсона/Спирмена |
| H2 | Доля аномальных закупок (единственный участник = победитель) выше в малых дочерних обществах | Все 10 компаний | χ²-тест, анализ доли |
| H3 | Закупки с нулевой НМЦ сконцентрированы в определённых категориях товаров/услуг | Поле «Наименование» + LLM-категоризация | Частотный анализ |
| H4 | Объём закупок ЮМани в сегменте ИТ-инфраструктуры вырос в 2025 г. относительно 2024 г. | Закупки ЮМани по годам | t-тест, визуализация |
